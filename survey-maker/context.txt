QUÉ SE CONSTRUYÓ EN ESTA SESIÓN
================================================================================

OBJETIVO
--------
Construir un generador de encuestas dinámico en Angular 21 con arquitectura
Signal-First y Zoneless, desplegado en GitHub Pages.

FASES COMPLETADAS
-----------------
1. Setup inicial
   - ng new survey-maker (Angular 21, standalone, routing, SCSS)
   - Zoneless: provideZonelessChangeDetection() en app.config.ts
   - Angular Material azure-blue (Material 3)
   - Reemplazar template default de Angular con <router-outlet />

2. Resource API (carga de encuestas)
   - SurveyLoaderService con rxResource (params/stream — API Angular 21)
   - parseSchema() — validador estructural puro, lanza en JSON inválido
   - Schemas JSON en public/schemas/ (servidos como assets estáticos)

3. Motor de renderizado (Form Engine)
   - FieldRendererComponent — dispatch por @switch (field.type)
   - 6 componentes de campo: text, number, textarea, select, radio, checkbox
   - Lógica condicional: isVisible computed() evalúa field.condition vs answers
   - SurveyComponent — orquesta secciones, progreso, submit
   - SurveyPageComponent — página routable con estados loading/error/success

4. Estado reactivo (Signals)
   - SurveyStateService — signal central de respuestas
   - progress computed() — ratio campos requeridos respondidos (0-100)
   - isValid computed() — todos los campos requeridos válidos
   - Validación: required, min/max (texto=longitud, número=valor), pattern
   - Persistencia localStorage via effect() con untracked()
   - touchAll() para marcar sucios antes de submit

5. Directivas
   - RequiredMarkDirective — agrega asterisco (*) rojo con Renderer2
   - FieldErrorDirective — toggle clase CSS via effect()

6. Tests (Vitest)
   - 35 tests en total
   - schema.parser.spec.ts — parseSchema() completo
   - survey-state.service.spec.ts — validación y estado
   - field-renderer.component.spec.ts — renderizado condicional

7. Documentación (docs/)
   - arquitectura.md
   - esquema-json.md
   - resource-api.md
   - motor-renderizado.md
   - signals.md
   - guia-uso.md

8. Despliegue GitHub Pages
   - ng build --base-href "/Survey-Maker/" (desde PowerShell, NO Bash)
   - npx angular-cli-ghpages --dir=dist/survey-maker/browser

9. Archivos de documentación y contexto (esta sesión)

================================================================================
ESQUEMA JSON DE ENCUESTA (RESUMEN)
================================================================================

{
  "id": "string (requerido)",
  "title": "string (requerido)",
  "description": "string (opcional)",
  "sections": [
    {
      "id": "string",
      "title": "string",
      "fields": [
        {
          "id": "string",
          "type": "text|number|textarea|select|radio|checkbox",
          "label": "string",
          "placeholder": "string (opcional)",
          "validation": {
            "required": boolean,
            "min": number,      // texto=longitud mínima, número=valor mínimo
            "max": number,      // texto=longitud máxima, número=valor máximo
            "pattern": "regex"
          },
          "options": [          // requerido para select/radio/checkbox
            { "value": "string", "label": "string" }
          ],
          "condition": {        // lógica condicional (opcional)
            "dependsOn": "field-id",
            "equals": "valor" | ["valor1", "valor2"]
          }
        }
      ]
    }
  ]
}

NOTAS IMPORTANTES PARA PRÓXIMA SESIÓN
================================================================================

1. SIEMPRE usar PowerShell para ng build --base-href, nunca Git Bash en Windows
2. angular-cli-ghpages maneja la rama gh-pages automáticamente — no tocar manual
3. Paths de HTTP deben ser RELATIVOS (sin / inicial) para respetar base-href
4. rxResource usa params/stream (Angular 21) — NO request/loader (API vieja)
5. Todos los componentes son standalone: true — no hay NgModules
6. Arquitectura Zoneless — NO usar setTimeout, setInterval directamente;
   usar signals y el scheduler de Angular
7. Para agregar campo nuevo: crear componente en features/fields/, agregar case
   en @switch de field-renderer.component.ts, agregar type en FieldType interface
8. Tests corren con Vitest: npm test (NO ng test)
